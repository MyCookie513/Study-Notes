如果本地仓库和云端仓库有不一致的地方会发生错误?
这时

git pull --rebase origin master

–rebase的作用是取消掉本地库中刚刚的commit，并把他们接到更新后的版本库之中。
总句：
1.先取消commit记录，并且把它们临时 保存为补丁(patch)(这些补丁放到”.git/rebase”目录中)，
2.之后同步远程库到本地，
3.最后合并补丁到本地库之中。
